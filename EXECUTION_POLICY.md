# EXECUTION POLICY — HormigasAIS Nodo-Escuela

## Purpose
This document defines the **execution boundaries** for all scripts, binaries,
agents, and procedures contained in this repository.

HormigasAIS Nodo-Escuela is a **sovereign training and validation environment**.
Execution is governed, contextual, and never implicit.

---

## Execution Contexts

### 1. LOCAL (Governed Execution)
- Scripts may only be executed by the **Master Mode Operator**.
- Execution requires an **active and valid GPS-based credential**.
- All actions are performed intentionally via terminal.
- No background, automatic, or implicit execution is permitted.

---

### 2. OFFLINE (Custodied Execution)
- Scripts may be executed offline **only under Master Mode supervision**.
- Execution must be explicitly validated in terminal using a valid GPS credential.
- Offline execution does **not** imply unrestricted use.
- Responsibility and traceability remain with the Master Operator.

---

### 3. EDUCATIONAL (Read-Only)
- **No script execution is permitted** in educational mode.
- Educational environments are strictly:
  - Read-only
  - Analytical
  - Observational
- This includes labs, sandboxes, demos, and educational mirrors
  such as TheAntsMind repositories.

---

## GitHub Execution Boundary

GitHub is **NOT** an execution environment for HormigasAIS.

- No script in this repository is intended to be executed on GitHub.
- No GitHub Actions, workflows, or automations are authorized
  to execute sovereign logic.
- GitHub serves exclusively as:
  - Code hosting
  - Documentation
  - Transparency interface
